# OS Design - 操作系统课程设计

## 项目定位
模拟真实的现代操作系统 | 本科生课设 | C++ | 教学为主

## 核心规范（每次会话必须遵守）

### 1. 难度控制
- 代码和示例对齐**本科生水平**，不过度工程化
- 每个新概念先解释原理，引导用户自己实现，而非直接代写

### 2. 开发流程
- **每次写新模块前，必须先全面检查项目**：
  - 编译通过无警告（`make clean && make`）
  - 无内存泄漏风险
  - 无逻辑错误、接口不一致
  - 无未完成的 TODO 或死代码
- 确认项目健康后，再进入新模块

### 3. 任务书遵循
- 严格遵循任务书的**功能目标**
- 具体实现择优选用，追求最优美、最清晰的方案
- 优美 = 简洁、合理、易理解、符合现代 C++ 实践

### 4. 教学方式
- 用户要**自己写代码**，AI 的角色是指导、审阅、答疑
